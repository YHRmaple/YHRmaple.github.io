使用hexo踩的一些坑

作者: 杨浩然 日期: July 10, 2019  11:14:40 分类: hexo

使用 hexo 搭建个人博客的教程有很多，基本上大同小异，这里我主要写一下自己在搭建过程中踩的一些坑。

1 关于 Deployer not found: git

解决方法：安装插件 hexo-deployer-git

之前把这个插件删了导致部署到 github 失败

2 关于 git 的一个错误

可尝试一下解决方法

1.升级 git

2.增加 git 缓冲区

3.更换网络

如果以上两种方法均不能解决问题，我建议换个网络试试。

这个错误是我在老家搭博客时遇到的，可能是老家网络的问题，我换了手机热点后解决了。。。

3 关于 Cannot GET /XXXX/

一般的解决方法是在创建的页面里更改类型，但我改了以后还是不行，无奈之下只好删除整个 blog 文件从零开始，在未做任何配置文件的改动时，页面可以正常显示，在我先后修改了 主题配置文件 和 站点配置文件 的语言（language）和时区（timezone）后，有出现了 cannot get 的错误，经过尝试，同时删除对 站点配置文件 的 language 和 timezone 修改后（两者保持默认），页面可以正常显示了，我也不知道这是什么原因（可能是两种设置冲突？）。

4 关于插入图片

直接将 \config.yml_ 中的 post_asset_folder 配置改为 true （注意，不需要安装 hexo-asset-image，我一开始安装后导致图片地址不对），之后在创建一个新文章是，会自动在_posts 文件夹下创建一个与文章名同名的文件夹，将所需图片放入该文件夹下，按照引入图片的格式引入即可（图片的地址直接写 图片名字.图片格式 ，前边不需要加上级文件夹即 XXX/ ）。
